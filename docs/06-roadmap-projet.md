# Roadmap projet

## Release 0 : cadrage et socle projet

Objectif : formaliser le besoin et preparer une base de travail commune.

Livrables attendus :

- documentation initiale du projet ;
- cadrage fonctionnel et non fonctionnel ;
- identification des acteurs ;
- description des processus metier ;
- liste des diagrammes a produire.

## Release 1 : authentification et utilisateurs

Objectif : mettre en place la gestion des comptes et des acces.

Livrables attendus :

- authentification ;
- gestion des utilisateurs ;
- gestion des roles ;
- controle d'acces initial ;
- premiers tests unitaires.

## Release 2 : departements et besoins

Objectif : permettre aux departements d'exprimer et suivre leurs besoins.

Livrables attendus :

- gestion des departements ;
- expression des besoins ;
- validation des besoins ;
- priorisation et suivi.

## Release 3 : fournisseurs et appels d'offres

Objectif : structurer l'acquisition des ressources.

Livrables attendus :

- gestion des fournisseurs ;
- creation des appels d'offres ;
- soumission des offres fournisseurs ;
- selection et attribution.

## Release 4 : ressources et inventaire

Objectif : construire le referentiel central des ressources materielles.

Livrables attendus :

- enregistrement des ressources ;
- categories et statuts ;
- suivi de l'inventaire ;
- recherche, filtrage et consultation.

## Release 5 : affectations

Objectif : gerer l'utilisation des ressources par les departements et utilisateurs.

Livrables attendus :

- affectation des ressources ;
- retour des ressources ;
- historique des affectations ;
- notifications associees.

## Release 6 : maintenance

Objectif : suivre les pannes, interventions et remplacements.

Livrables attendus :

- signalement de panne ;
- diagnostic ;
- intervention de maintenance ;
- retour fournisseur ;
- reparation ou remplacement.

## Release 7 : tests, securite et qualite

Objectif : renforcer la fiabilite et la qualite du projet.

Livrables attendus :

- tests unitaires et tests d'integration ;
- couverture des regles metier critiques ;
- verification des droits d'acces ;
- qualite de code ;
- corrections issues des tests.

## Release 8 : CI/CD, deploiement et rapport final

Objectif : automatiser les controles et finaliser le projet.

Livrables attendus :

- pipeline CI/CD ;
- controles automatises ;
- strategie de deploiement ;
- documentation finale ;
- rapport de synthese et bilan pedagogique.
