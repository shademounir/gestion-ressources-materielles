# Acteurs et roles

## Administrateur

L'administrateur gere la configuration globale de l'application. Il cree les comptes, attribue les roles, supervise les droits d'acces et veille au bon fonctionnement general du systeme.

Responsabilites principales :

- gerer les utilisateurs et leurs permissions ;
- parametrer les referentiels de base ;
- superviser les journaux d'audit ;
- intervenir en cas de probleme d'acces ou de configuration.

## Responsable des ressources

Le responsable des ressources pilote l'inventaire et le cycle de vie des biens materiels de la faculte.

Responsabilites principales :

- enregistrer les ressources recues ;
- tenir a jour l'inventaire ;
- suivre les etats des ressources ;
- gerer les affectations et retours ;
- coordonner les maintenances et remplacements.

## Chef de departement

Le chef de departement exprime, consolide et valide les besoins materiels de son departement.

Responsabilites principales :

- declarer ou valider les besoins du departement ;
- prioriser les demandes ;
- consulter les ressources affectees au departement ;
- suivre l'avancement des demandes.

## Enseignant

L'enseignant peut exprimer un besoin materiel, demander une ressource ou signaler une panne sur un equipement utilise.

Responsabilites principales :

- formuler des demandes de ressources ;
- signaler les pannes ;
- consulter l'etat des demandes le concernant ;
- restituer les ressources selon les procedures definies.

## Fournisseur

Le fournisseur repond aux appels d'offres et peut etre sollicite pour des retours, reparations ou remplacements.

Responsabilites principales :

- consulter les appels d'offres ouverts ;
- soumettre des offres ;
- fournir les ressources selectionnees ;
- assurer les garanties, reparations ou remplacements prevus.

## Technicien maintenance

Le technicien maintenance prend en charge les incidents techniques et assure le suivi des interventions.

Responsabilites principales :

- analyser les signalements de panne ;
- diagnostiquer les ressources defectueuses ;
- realiser ou suivre les reparations ;
- renseigner l'historique de maintenance ;
- proposer un retour fournisseur ou un remplacement si necessaire.

## Systeme de notification

Le systeme de notification informe les acteurs des evenements importants du processus.

Responsabilites principales :

- notifier les validations ou refus ;
- alerter sur les nouvelles demandes ;
- signaler les changements d'etat ;
- rappeler les actions en attente ;
- informer les utilisateurs concernes par une affectation, une maintenance ou une livraison.
